USER STORIES FOR STUDENT INTERN

1.As a student, I want to register/login to the system so that I can access my internship dashboard

2.As a student, I want to submit my internship placement details so that the coordinator can approve my placement

3.As a student, I want to create weekly log entries in draft mode so that I can save my progress before final submission

4.As a student, I want to submit my weekly log for supervisor review so that my progress can be evaluated

5.As a student, I want to view feedback comments from my industry supervisor so that I can improve my performance

6.As a student, I want to edit and resubmit rejected logs with corrections so that my supervisor can approve them

7.As a student, I want to view my submitted logs history so that I can track my progress throughout the internship

8.As a student, I want to receive notifications when my log is approved, rejected, or when deadlines approach

9.As a student, I want to view my final weighted scores and feedback after evaluation so that I know my performance outcome

10.As a student, I want to upload supporting documents/images with my weekly logs so that I can provide evidence of my work

11.As a student, I want to be prevented from submitting logs after the deadline so that I follow the internship rules

REQUIREMENTS FOR STUDENT INTERN

1.The system shall support registration and login for all user roles

1.The system shall implement role-based access control where users can only access features permitted for their role

1.The system shall support password reset functionality via email

2.The system shall allow students to submit internship placement details (company name, address, start date, end date, supervisor name/email)

2.The system shall validate that placement dates do not overlap with existing approved placements for the same student

3.The system shall allow students to create weekly log entries with fields: week number, date, tasks completed, hours worked, skills learned, challenges, attachments

3.The system shall allow students to save logs as draft without submitting for review

4.The system shall allow students to submit draft logs for supervisor review, changing state to 'Submitted'

3&4&6.The system shall maintain log states: Draft, Submitted, Rejected, Approved

5.The system shall notify the industry supervisor when a new log is submitted for their review

6.The system shall enforce valid state transitions: Draft→Submitted, Submitted→Approved, Submitted→Rejected, Rejected→Submitted

6.The system shall allow students to edit and resubmit rejected logs (Rejected→Submitted)

7.The system shall allow students to view all their logs with their current status

8.The system shall send reminder notifications to students 24 hours before log submission deadlines

8.The system shall change placement status from 'Pending' to 'Active' upon approval, and to 'Completed' after end date

8.The system shall prevent students from submitting logs until their placement is approved

9&7.The system shall provide a student dashboard showing: active placement, logs status (pending/approved/rejected), deadlines, and final results when available

9.The system shall display the final score and grade to students once all evaluations are complete

9.The system shall compute and store a final grade,"e.g., A, B, C" based on the weighted score and institutional grading scale

9.The system shall automatically calculate the final weighted score based on configured formula

11.The system shall enforce submission deadlines based on configured rules

11.The system shall prevent submission of logs after the deadline (logs become locked)
